# [DeepNatural] 프론트엔드 코딩 테스트

## 실행

```bash
# 설치
yarn install

# devServer 실행
yarn dev

# http://localhost:8080
```

## 문제

1. 레이블링 도구가 웹브라우저에 띄워지면, 도구는 서버로부터 작업 데이터를 받아옵니다. (taskAssign API)
2. 받아오는 작업 데이터는 아래의 내용으로 구성됩니다:
    - 화면 좌측 패널의 작업 가이드 내용 (제목과 본문)
    - 화면 중앙의 대화 내용
    - 화면 우측 패널의 내용 (패널의 제목, 설명, 그리고 레이블 내용)
3. 하나의 대화모음에는 두 명의 화자가 등장합니다. 그 중 하나의 화자의 발화들에 대해서만 레이블링을 수행합니다.
    (어느 화자에 대해 레이블링할지는 JSON 데이터에 기재되어 있습니다.)
4. 레이블링할 대화 말풍선을 클릭하고 우측의 레이블 후보 중 하나를 클릭하면, 해당 대화 말풍선과 선택된 레이블의 색이 (하늘색으로) 바뀌고, 대화 말풍선의 커서가 다음 레이블링 가능한 대화 말풍선으로 이동합니다.
5. 레이블링이 진행됨에 따라, 좌측 패널의 진행바가 갱신됩니다.
6. 이미 레이블링한 기존의 대화 말풍선을 클릭하면, 작업자가 선택했었던 레이블을 우측 패널의 레이블 카테고리에서 보여줍니다.
    그 상태에서 기존에 선택된 레이블을 변경하여 선택할 수 있습니다. (이 경우에는 대화 말풍선의 커서가 다음 레이블 가능한 대화 말풍선으로 이동하지 않고 현재 말풍선에 머무릅니다.)
7. 작업자는 화면 하단 도구상자에서 '이전', '다음' 버튼을 클릭하여, 레이블링 대상 대화 말풍선의 커서를 앞뒤로 이동할 수 있습니다.
8. 마지막 대화까지 레이블링이 끝나면, 작업자는 화면 하단 도구상자에서 '작업완료' 버튼을 클릭합니다. 이 때 대화 레이블링 정보를 서버로 전송합니다. (taskResult API).
    그리고 크레딧 포인트를 1만점 증가시킵니다. 그리고 곧바로 다음 작업 데이터를 받아와서 다음 레이블링을 시작합니다.

작업도구 개발 시 고려할 사항입니다.

1. 실제로는 서버에 HTTP Web API를 요청하여 JSON 형태로 데이터를 받아오겠지만, 이번 과제에서는 프론트엔드에서 mockup JSON을 임의로 정의하여 사용합니다.
2. 웹브라우저로는 크롬과 파이어폭스 최신 버전을 사용하여 확인합니다.
3. 주요 화면 크기는, 800~1440px 사이에서 동작하는 것을 목표로 합니다.
4. 해당 작업도구의 화면을 Vue.js 기반의 웹 애플리케이션으로 구현합니다.
5. UI Framework 는 제한없이 자유롭게 사용하셔도 됩니다.
